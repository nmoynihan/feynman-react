ASSISTANT
Identified two main causes of sluggishness:

1. The demo was serializing the full diagram JSON into the textarea on every drag frame.
2. The editor was emitting full controlled updates on every pointer event.

Implemented performance and UX fixes:

- Drag updates in packages/feynman-editor/src/index.tsx are now throttled to requestAnimationFrame.
- Auto-fit viewport updates no longer fight active drags.
- Expensive custom label rendering is skipped during active drags to reduce frame cost.
- Demo-side JSON synchronization in apps/demo/src/App.tsx is now debounced instead of reserializing on every drag frame.
- Add-edge mode now creates vertices automatically when the user clicks empty space.
  - If the first click in add-edge mode is on empty space, a source vertex is created.
  - If the second click is on empty space, a target vertex is created and the edge is committed.

Also fixed lint configuration issues so repo linting became clean again.
